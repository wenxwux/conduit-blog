Claude Code をインストールしたあと、本当につまずくのは使い方ではなく、たいてい**接続**です。キーをどこに入れるのか、BASE URL をどう設定するのか、なぜ実行するとエラーになるのか。本記事では、私自身が遭遇し、また質問されることも多かった代表的なつまずきポイントをまとめて解説します。手順どおりに確認すれば、ほとんどのケースは解決できます。

## 一、まず接続に何が必要かを押さえる

Claude Code が動作するのに必要なのは 2 つだけです。

- **API キー 1 つ**(本人確認のための資格情報)
- **BASE URL 1 つ**(リクエストの送信先)

もっとも一般的なのは環境変数に書く方法です。

```bash
export ANTHROPIC_AUTH_TOKEN="あなたのキー"
export ANTHROPIC_BASE_URL="あなたの接続先アドレス"
```

> この 2 行は `~/.zshrc` または `~/.bashrc` に書いておくことをおすすめします。書き換えたら `source ~/.zshrc` を実行して反映させれば、以降はターミナルを開くたびに自動で読み込まれ、毎回手動で export する必要がなくなります。

## 二、よくあるエラー一覧

### 1. `401 Unauthorized`(認証失敗)

もっとも多いケースです。基本的にはキーの問題です。

- キーをコピーするときに余分なスペースや改行が入った → きれいにコピーし直す
- 環境変数名を間違えている → `ANTHROPIC_AUTH_TOKEN` であることを確認
- 変数を変えたのに反映されていない → ターミナルを開き直すか、`source` する
- そもそも読み込めているか確認:

```bash
echo $ANTHROPIC_AUTH_TOKEN
```

出力が空なら、設定できていません。

### 2. 接続タイムアウト / 無応答

たいていは BASE URL の問題です。

- アドレスのスペルミス → 1 文字ずつ照合する
- **末尾にスラッシュが余分に付いている** → これは頻出のつまずきです。`https://xxx.com/` 末尾の `/` を外してみる
- プロトコルが `http` になっている → 通常は `https` を使う
- 現在の値を確認:

```bash
echo $ANTHROPIC_BASE_URL
```

### 3. `404 Not Found`

多くは BASE URL のパスが正しくないケースです。BASE URL は通常**ドメインのルートまで**(または接続手順に応じて `/v1` まで)を入れるだけで、完全な API パスまで書き込んではいけません。残りのパスは Claude Code が自動で補完します。

### 4. モデルが存在しない / `model not found`

指定したモデル名が、接続先でサポートされていないか、スペルミスです。対処法:

- 接続先がサポートする標準的なモデル名を使う
- Claude Code 内で `/model` を使って選択肢を確認する
- タイプミス(バージョン番号の誤りなど)がないか確認する

### 5. 設定を変えたのに反映されない

このつまずきは非常に多いので、独立して挙げておきます。

- 環境変数を変更したら、**必ずターミナルを開き直すか source する**こと。そうしないと現在のセッションには反映されません
- 複数の場所(環境変数、設定ファイル)で同時に設定していると、互いに上書きし合うことがあります。1 か所に絞りましょう
- どうしても混乱したら、関連する変数を `unset` でクリアして設定し直します:

```bash
unset ANTHROPIC_AUTH_TOKEN
unset ANTHROPIC_BASE_URL
```

そのうえで改めて export します。

## 三、トラブル確認の早見表

| エラー症状 | 多くの原因 | 対処法 |
|---|---|---|
| 401 Unauthorized | キーが間違い / 未反映 | キーを入れ直し、環境変数を確認 |
| 接続タイムアウト | BASE URL が間違い / スラッシュ余分 | アドレスを照合し、末尾スラッシュを外す |
| 404 Not Found | パスを書きすぎ | BASE URL はルートか /v1 まで |
| model not found | モデル名が間違い | サポートされるモデル名を使い、`/model` で確認 |
| 変えても反応なし | 環境変数を再読み込みしていない | ターミナルを開き直すか source |

## 四、万能なトラブル確認の手順

どんな接続エラーでも、私はいつもこの順番で進めます。これが何度試しても効きます。

1. `echo` でキーと BASE URL が**本当に読み込めているか**確認する
2. BASE URL の**末尾にスラッシュが余分にないか**、プロトコルが正しいかを確認する
3. **ターミナルを開き直して**、環境変数が未反映の可能性を排除する
4. もっとも基本的な指示を 1 つ試して、問題の範囲を絞り込む

## まとめ

接続の本質は 2 つだけです。**キーを正しく入れること、BASE URL を正しく設定すること**。9 割のエラーは「キーが未反映」と「BASE URL の誤り」のどちらかに収まります。まず `echo` で確認、次にスラッシュをチェック、最後にターミナルを開き直す。この順番を覚えておけば、たいてい自分で解決できます。設定さえ済めば、Claude Code はあなたのコードを読み、指示どおりに自動で編集してくれる頼れる相棒になります。
