多くの人にとって、Claude Code を始めるときの難所は「使い方」ではなく、**つなぐこと**にあります。キーをどこに入れるのか、BASE URL はどう設定するのか、なぜ最初の実行でエラーになるのか。本記事では一連の流れをひととおり追っていくので、約 5 分で AI にコードを編集させられるようになります。

## 1. Claude Code をインストールする

Claude Code は CLI ツールです。インストールはコマンド 1 つで済みます。

```bash
npm install -g @anthropic-ai/claude-code
```

実行後に `claude --version` を打って、バージョン番号が表示されれば成功です。

## 2. アクセスを設定する(核心のステップ)

Claude Code が動くために必要なのは 2 つだけです。**API キー**と **BASE URL** です。

もっとも一般的な方法は環境変数で設定することです。

```bash
export ANTHROPIC_AUTH_TOKEN="あなたのキー"
export ANTHROPIC_BASE_URL="あなたの接続先URL"
```

> ヒント:この 2 行を `~/.zshrc` または `~/.bashrc` に書いておくと、新しいターミナルを開くたびに自動で読み込まれます。

## 3. 初めての AI コード編集

任意のプロジェクトディレクトリに入って、次を実行します。

```bash
claude
```

あとはやりたいことを普通の言葉で伝えるだけです。たとえば:

> 「utils.js の中の `var` をすべて `const` に変更して、短いコメントを付けて。」

Claude Code はファイルを読み、変更案を提示し、あなたが確認したら書き戻します。あなたの仕事はレビューして承認することだけです。

## 4. よくあるエラー

| 症状 | 考えられる原因 | 対処法 |
|---|---|---|
| `401 Unauthorized` | キーが間違っている、または未設定 | キーの値と、環境変数が export されているか確認 |
| タイムアウト / 無応答 | BASE URL の誤りまたはネットワークの問題 | URL のスペルを確認、末尾にスラッシュを付けない |
| Model not found | モデル名が間違っている | 接続先がサポートするモデル名を使う |

## まとめ

つなぐことの本質は、結局 2 つに集約されます。**正しいキー**と**正しい BASE URL** です。これらさえ設定できれば、Claude Code はあなたのコードを読み、指示どおりに編集してくれるアシスタントになります。まずは小さな変更で確認フローに慣れ、それから大きなタスクを任せていきましょう。
