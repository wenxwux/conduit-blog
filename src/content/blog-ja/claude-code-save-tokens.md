Claude Code でコードを書くのは本当に快適ですが、長く使っていると気づきます。token の消費もまた本当に速い、と。大きなプロジェクトで数ターンやり取りするだけで、コンテキストはどんどん積み上がります。本記事は、私が実際に試して確かに効果があった 7 つの token 節約テクニックです。おまじないではなく、どれもすぐに実践できます。

## 一、`/clear` でこまめにコンテキストをクリアする

これはもっとも見落とされがちで、もっとも効果的な一手です。Claude Code はデフォルトで会話履歴全体を付けてリクエストを送ります。会話が長くなるほど、毎ターン送る token が増えます。たとえそのあとに無関係な新しい質問をしても、です。

タスクを切り替えるときは、すぐに:

```bash
/clear
```

1 つのタスクに 1 つの会話。今の私の習慣は、機能を 1 つ直し終えたら一度クリアすること。無関係な履歴をずっとぶら下げてコストを積み上げさせません。

## 二、`.claudeignore` で無関係なファイルを除外する

Claude Code はプロジェクトを読むときに大量のファイルをスキャンしますが、その多くはモデルに渡す必要のないもの——依存ディレクトリ、ビルド成果物、ログなどです。`.claudeignore` を作りましょう(構文は `.gitignore` と同じです)。

```
node_modules/
dist/
build/
*.log
.next/
coverage/
```

こうしておけば、スキャンや検索のときに自動でスキップされ、速くて節約にもなります。

## 三、読むファイルを明示し、「全体検索」をさせない

あいまいな指示は広範囲なスキャンを引き起こします。比べてみましょう。

> ❌ 「プロジェクトでログインがどう実装されているか見て」
>
> ✅ 「`src/auth/login.ts` を読んで、この関数のロジックを教えて」

2 つ目はファイルを直接特定しているので、大きな検索コストを省けます。自分がどこを直したいかが明確なほど、token の消費は少なくなります。

## 四、CLAUDE.md をしっかり書いて、説明の繰り返しを減らす

毎回「このプロジェクトは TypeScript を使う、パッケージ管理は pnpm、テストは vitest」と説明する……こうした繰り返しの説明は、プロジェクトルートの `CLAUDE.md` に一度書いてしまえます。モデルが自動で読み取るので、毎ターン背景を説明し直す必要がなくなります。

CLAUDE.md をどう書けば使いやすいかは、[使える CLAUDE.md の書き方](/conduit-blog/ja/blog/claude-md-guide/)という記事を別に書いたので、あわせて読んでください。

## 五、小さなタスクは Sonnet、なんでも Opus を使わない

すべてのタスクに最強モデルが必要なわけではありません。日常の小さな修正、コメント追加、テスト作成なら Sonnet で十分ですし、単価もずっと安いです。本当に複雑な推論や大規模リファクタリングが必要なときだけ Opus に切り替えましょう。

切り替えは簡単です。

```bash
/model
```

適切なモデルを選ぶこと自体が、もっとも大きな節約になります。場面ごとの具体的な選び方は、[Opus と Sonnet の選び方](/conduit-blog/ja/blog/opus-vs-sonnet/)を参照してください。

## 六、長いタスクは小さなステップに分け、一気に詰め込まない

「プロジェクト全体を一度にリファクタリングして」と一気に投げると、膨大なファイルを読み、超長文の回答を生成し、token が爆発するうえに脱線もしやすくなります。小さなステップに分けましょう。

1. まずリファクタリング計画を出させる(計画だけ出力させ、コードは触らせない)
2. あなたが確認したら、モジュールを 1 つずつ直していく
3. 1 つ直し終えるたびに `/clear` して次に進む

ステップ分割は token を節約するだけでなく、品質も明らかに安定します。

## 七、キャッシュを活用する:繰り返すコンテキストを何度も送らない

同じ会話の中で同じファイル群を繰り返し参照する場合、Claude のプロンプトキャッシュ機構によって、繰り返し部分が非常に安くなります。コツは**安定したコンテキスト構造を保つこと**——変わらない背景情報を前に、変化する質問を後ろに置くと、キャッシュのヒット率が上がります。

## まとめ

token 節約の核心は一言です。**モデルには本当に必要なものだけを渡す**。こまめに履歴をクリアし、無関係なファイルを除外し、ファイルを正確に指定し、適切なモデルを選び、大きなタスクを分割する。これらを組み合わせると、私自身の日常の消費はおよそ半分以上減り、効果はほとんど落ちませんでした。
