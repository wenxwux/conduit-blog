Cursor をしばらく使ってみて、「まあこんなものか」と思っていませんか？チャットして Tab で補完するだけ？実は、その能力の 30% しか使っていないかもしれません。この記事では見落としがちな 10 の隠れテクニックを紹介します。ツールを変える必要もプラグインを入れる必要もなく、現行バージョンですぐに使えます。

## テクニック 1：4層の Rules システムを使いこなす（.cursorrules だけじゃない）

多くの人はプロジェクトルートに `.cursorrules` ファイルを置くことしか知りませんが、Cursor には実際に**4層の Rules システム**があり、優先度は高い方から：

| 層 | 設定場所 | 適用範囲 |
|------|----------|----------|
| Team Rules | Cursor バックエンド設定 | チーム全体 |
| Project Rules | `.cursor/rules/*.mdc` | 現プロジェクト |
| User Rules | Cursor Settings → Rules | すべてのプロジェクト |
| AGENTS.md | プロジェクトルートまたはサブディレクトリ | 対応ディレクトリ範囲 |

重要操作：`.cursor/rules/` ディレクトリに `.mdc` ファイルを作成し（注意：`.mdc` であって `.md` ではない、後者は無視されます）、frontmatter で有効条件を制御します：

```yaml
---
description: "React コンポーネント開発規約"
globs: "src/**/*.tsx, src/**/*.jsx"
alwaysApply: false
---
コンポーネントは必ず TypeScript + 関数型で書くこと……
```

4つの有効モードを柔軟に組み合わせ可能：`alwaysApply: true`（常にロード）、`description` のみ記述（Claude が自動判断でロード）、`globs` 記述（ファイルマッチ時に自動付加）、何も書かない（手動で `@rule-name` を参照した時のみロード）。

**ショートカット**：Agent チャットで `/create-rule` を入力すると、対話形式でルールを作成できます。

## テクニック 2：Tab 補完を単語単位で受け入れる

ほとんどの人は Tab で全部受け入れるか、Escape で全部拒否するかの二択ですが、中間の選択肢もあります：

- **`Cmd + →`**（Mac）/ **`Ctrl + →`**（Windows）：単語単位で補完を受け入れ

AI の補完の前半は正しいけど後半がイマイチな時、前半の正しい部分だけ単語単位で受け入れてから自分で続きを入力する方が、再生成よりずっと効率的です。

## テクニック 3：Tab の「ジャンプ予測」とクロスファイル補完

これは Tab の最も見落とされがちな機能——**補完を受け入れた後にもう一度 Tab を押す**：

1. Cursor が**次の編集位置を予測**して自動的にカーソルをジャンプ
2. 変更が別のファイルに関係する場合、エディタ下部に**ポータルウィンドウ**が表示され、クロスファイルの修正候補を表示

例えば関数シグネチャを変更した後、Tab が自動的にその関数の呼び出し箇所にジャンプし、パラメータの同期修正を提案します。大量の手動検索とジャンプが不要に。

## テクニック 4：Agent 動作中にメッセージをキューイング

Agent がタスクを実行中でも、待つ必要はありません——そのまま**後続の指示を入力**でき、キューに入って順番に実行されます：

| 操作 | ショートカット | 効果 |
|------|--------|------|
| キューイング | `Enter` | 現タスク完了後に順番に処理 |
| 即時挿入 | `Cmd + Enter` | 現メッセージに追加、緊急修正用 |
| 順序調整 | メッセージをドラッグ | 優先順位を並べ替え |

緊急シナリオ：Agent が触ってほしくないファイルを変更中に、`Cmd + Enter` で「止まれ、そのファイルに触るな！」を送信——即座に読み取られます。

## テクニック 5：デザインモード——ブラシで UI 修正を指示

Cursor 内蔵ブラウザには**デザインモード**（Design Mode）があり、UI の調整に最適です：

1. Cursor 内蔵ブラウザを開き、実行中のアプリに移動
2. ページ上で直接**クリック・矩形選択・丸囲み**して変更したい箇所をマーク
3. Agent がマークを読み取り、自動でコードを修正

複数要素を選択すると、Cursor はレイアウト関係を理解し、全体的に調整します。音声入力と組み合わせれば、指差しながら話し、Agent が1つの変更を完了したらすぐ次の口述要求を処理します。

## テクニック 6：クラウドサブエージェント /in-cloud

Agent にローカルリソースを使わせたくない場合は `/in-cloud` でタスクをクラウドに投げます：

```text
/in-cloud auth モジュールをリファクタリングして JWT refresh token ロジックを追加して
```

クラウド Agent は独立した VM 上で動作し、自動でブランチを作成、完了後に PR を提出します。**複数のクラウド Agent を同時に並列実行**できます。

さらに：`/babysit` コマンドでクラウド Agent に PR がマージ可能になるまで自走させることができます——テスト実行、問題修正、コード更新、すべて自動。

**チーム協力ヒント**：リポジトリに `.cursor/environment.json` をコミットすると、クラウド Agent がプリセット環境設定（依存関係、環境変数など）を使用し、チーム全体の恩恵になります。

## テクニック 7：コミット前の /review セルフチェック

コード完成後に品質が不安？プッシュ前に内蔵レビューを使いましょう：

```text
/review              # 完全レビュー（バグ検出 + セキュリティ監査）
/review-bugbot       # バグ検出のみ
/review-security     # セキュリティ監査のみ
```

Bugbot はローカルで完了後、同じ diff を GitHub にプッシュすると自動で既にレビュー済みと認識し、重複チェックをスキップして PR 内でローカルレビュー結果を引用します。

## テクニック 8：MCP 変数補間——サーバー起動なしでカスタムツールを使う

`mcp.json` 設定では環境変数の補間がサポートされていますが、知らない人が多いです：

```json
{
  "mcpServers": {
    "my-server": {
      "command": "node",
      "args": ["${workspaceFolder}/tools/server.js"],
      "env": {
        "API_KEY": "${env:MY_API_KEY}",
        "HOME": "${userHome}"
      }
    }
  }
}
```

使用可能な3つの変数：
- `${env:変数名}` — システム環境変数から読み取り
- `${userHome}` — ユーザーのホームディレクトリ
- `${workspaceFolder}` — 現プロジェクトのルートディレクトリ

MCP のデバッグで問題が発生した場合、`Cmd + Shift + U` で Output パネルを開き → **MCP Logs** ドロップダウンを選択すると、完全な送受信ログとエラー情報を確認できます。

## テクニック 9：ファイルタイプ別に Tab 補完をオフにする

Markdown、JSON、YAML を書く時に Tab 補完が邪魔？**ファイルタイプ別にオフ**にできます：

1. エディタ右下の Cursor Tab ステータスインジケーターをクリック
2. **Disable for specific extensions** を選択
3. `.md`、`.json`、`.yaml` などを追加

さらに Tab 機能を**一時停止**することも可能——時間を選択（例：30分）すると、集中コーディング中に邪魔されず、時間になったら自動で復帰します。

## テクニック 10：Checkpoints——Git より細かい粒度のタイムトラベル

Agent が大きな変更を行うたびに、Cursor は自動で **Checkpoint スナップショット**を撮ります：

- Agent パネルの任意の checkpoint をクリックし、**その時点のすべてのファイル状態をプレビュー**
- **Restore Checkpoint** をクリックすると、その状態に完全ロールバック
- checkpoint にホバーして **+** をクリックすると、**その地点から新しいブランチを作成**して開発を続行

Checkpoint はローカルに保存され、**Git とは独立**——まだ commit していなくても遡れます。Agent に大胆なリファクタリングをさせる時に特に安心感があります：自由に変更させて、壊れたらワンクリックでロールバック。

## ショートカット早見表

| 操作 | Mac | Windows |
|------|-----|---------|
| Agent チャットを開く | `Cmd + I` | `Ctrl + I` |
| 単語単位で Tab 補完を受け入れ | `Cmd + →` | `Ctrl + →` |
| コマンドパレット | `Cmd + Shift + P` | `Ctrl + Shift + P` |
| 即時送信（キューをスキップ） | `Cmd + Enter` | `Ctrl + Enter` |
| Output パネル（MCP デバッグ） | `Cmd + Shift + U` | `Ctrl + Shift + U` |
| Cursor 設定 | `Cmd + Shift + J` | `Ctrl + Shift + J` |
