Claude Code를 설치한 뒤 정말로 사람을 막히게 하는 건 사용법이 아니라 **접속**인 경우가 많습니다. 키를 어디에 넣는지, BASE URL을 어떻게 설정하는지, 왜 실행하자마자 오류가 나는지 말이죠. 이 글에서는 제가 직접 겪었고, 또 가장 많이 질문받은 함정들을 한데 모아 설명합니다. 순서대로 점검하면 거의 다 해결할 수 있습니다.

## 1. 접속에 무엇이 필요한지부터 파악하기

Claude Code가 작동하는 데는 단 두 가지만 필요합니다.

- **API 키 하나** (신원 인증)
- **BASE URL 하나** (요청을 어디로 보낼지)

가장 흔한 방식은 환경 변수에 작성하는 것입니다.

```bash
export ANTHROPIC_AUTH_TOKEN="너의key"
export ANTHROPIC_BASE_URL="너의접속주소"
```

> 이 두 줄을 `~/.zshrc` 또는 `~/.bashrc`에 작성한 뒤 `source ~/.zshrc`를 실행해 적용하는 것을 권장합니다. 이후로는 터미널을 열 때마다 자동으로 불러오므로, 매번 수동으로 export 할 필요가 없습니다.

## 2. 자주 발생하는 오류 모음

### 1. `401 Unauthorized` (인증 실패)

가장 흔합니다. 기본적으로 키 문제입니다.

- 키를 복사할 때 공백이나 줄바꿈이 섞임 → 깨끗하게 다시 복사
- 환경 변수 이름을 잘못 씀 → `ANTHROPIC_AUTH_TOKEN`이 맞는지 확인
- 변수를 바꿨지만 적용되지 않음 → 터미널을 다시 열거나 `source` 실행
- 실제로 읽히고 있는지 검증:

```bash
echo $ANTHROPIC_AUTH_TOKEN
```

출력이 비어 있으면 설정되지 않은 것입니다.

### 2. 연결 타임아웃 / 응답 없음

보통 BASE URL 문제입니다.

- 주소 철자가 틀림 → 한 글자씩 대조
- **끝에 슬래시가 더 붙음** → 자주 나오는 함정입니다. `https://xxx.com/` 끝의 `/`를 빼고 시도해 보세요.
- 프로토콜을 `http`로 씀 → 보통 `https`를 써야 합니다
- 현재 값 검증:

```bash
echo $ANTHROPIC_BASE_URL
```

### 3. `404 Not Found`

대부분 BASE URL 경로가 잘못된 경우입니다. BASE URL은 보통 **도메인 루트까지만**(또는 접속 안내에 따라 `/v1`까지만) 작성하며, 전체 API 경로까지 넣지 마세요. Claude Code가 뒤쪽 경로를 자동으로 채웁니다.

### 4. 모델 없음 / `model not found`

지정한 모델 이름을 접속 측에서 지원하지 않거나 철자가 틀린 경우입니다. 해결:

- 접속 제공자가 지원하는 표준 모델 이름 사용
- Claude Code에서 `/model`로 선택 가능한 옵션 확인
- 오타가 없는지 확인 (예: 버전 번호 오기)

### 5. 설정을 바꿨는데 적용되지 않음

너무 흔한 함정이라 따로 떼어 설명합니다.

- 환경 변수를 바꾼 뒤에는 **반드시 터미널을 다시 열거나 source** 해야 현재 세션에 반영됩니다
- 여러 곳에 동시에 설정했다면(환경 변수, 설정 파일) 서로 덮어쓸 수 있으니, 한 곳만 남기세요
- 정말 꼬였다면 관련 변수를 `unset`으로 지우고 다시 설정:

```bash
unset ANTHROPIC_AUTH_TOKEN
unset ANTHROPIC_BASE_URL
```

그런 다음 다시 export 하세요.

## 3. 점검용 빠른 참조표

| 오류 증상 | 대부분의 원인 | 해결법 |
|---|---|---|
| 401 Unauthorized | 키 오류 / 미적용 | 키 재입력, 환경 변수 확인 |
| 연결 타임아웃 | BASE URL 오류 / 슬래시 과다 | 주소 대조, 끝 슬래시 제거 |
| 404 Not Found | 경로를 너무 많이 씀 | BASE URL은 루트나 /v1까지만 |
| model not found | 모델 이름 오류 | 지원 모델 이름 사용, `/model`로 확인 |
| 바꿨는데 반응 없음 | 환경 변수 미재로드 | 터미널 재시작 또는 source |

## 4. 만능 점검 순서

어떤 접속 오류든 저는 이 순서대로 처리하며, 늘 효과가 있습니다.

1. `echo`로 키와 BASE URL이 **실제로 읽히는지** 확인
2. BASE URL **끝에 슬래시가 더 붙지 않았는지**, 프로토콜이 맞는지 점검
3. **터미널을 다시 열어** 환경 변수 미적용 가능성 배제
4. 가장 기본적인 명령을 하나 시도해 문제 범위 좁히기

## 마무리

접속의 본질은 두 가지입니다. **키를 올바로 넣고, BASE URL을 올바로 설정하는 것**입니다. 오류의 90%는 "키 미적용"과 "BASE URL 오기" 두 부류를 벗어나지 않습니다. 먼저 `echo`로 검증하고, 슬래시를 확인한 뒤, 마지막으로 터미널을 다시 여는 순서를 기억하면 대부분 스스로 해결할 수 있습니다. 설정을 마치면 Claude Code는 여러분의 코드를 읽고 명령에 따라 자동으로 수정해 주는 든든한 조력자가 됩니다.
