Claude Code로 코드를 쓰면 정말 쾌적합니다. 하지만 오래 쓰다 보면 token이 정말 빠르게 타들어 간다는 걸 알게 됩니다. 큰 프로젝트에서 몇 차례만 대화해도 컨텍스트가 금세 쌓이죠. 이 글은 제가 직접 실측해서 확실히 효과가 있었던 token 절약 기법 7가지입니다. 미신이 아니라, 하나하나 바로 적용할 수 있습니다.

## 1. `/clear`로 컨텍스트를 제때 비우기

가장 간과되면서도 가장 효과적인 방법입니다. Claude Code는 기본적으로 전체 대화 기록을 함께 보내 요청합니다. 오래 대화할수록 매 차례 전송해야 할 token이 늘어나죠. 나중에 묻는 질문이 전혀 관련 없는 새 질문이어도 마찬가지입니다.

작업을 전환할 때는 바로:

```bash
/clear
```

작업 하나에 대화 한 단락. 지금 제 습관은 기능 하나를 다 고치면 한 번 비우는 것입니다. 무관한 기록이 계속 매달려 비용으로 누적되게 절대 두지 않습니다.

## 2. `.claudeignore`로 무관한 파일 차단하기

Claude Code는 프로젝트를 읽을 때 수많은 파일을 스캔하는데, 그중 상당수는 모델에 먹일 필요가 없습니다. 예를 들어 의존성 디렉터리, 빌드 산출물, 로그 같은 것들이죠. `.claudeignore`를 하나 만드세요(문법은 `.gitignore`와 동일합니다).

```
node_modules/
dist/
build/
*.log
.next/
coverage/
```

이렇게 하면 스캔과 검색 시 자동으로 건너뛰어 빠르고 절약됩니다.

## 3. 읽을 파일을 명확히 지정하고, "전역 검색"을 시키지 말기

모호한 지시는 대규모 스캔을 유발합니다. 비교해 보세요.

> ❌ "프로젝트에서 로그인이 어떻게 구현됐는지 봐줘"
>
> ✅ "`src/auth/login.ts`를 읽고 이 함수의 로직을 알려줘"

두 번째는 파일을 바로 지목해, 큰 검색 비용을 절약합니다. 어디를 고쳐야 하는지 분명할수록 token을 적게 씁니다.

## 4. CLAUDE.md를 잘 써서 반복 설명 줄이기

매번 "이 프로젝트는 TypeScript를 쓰고, 패키지 관리는 pnpm, 테스트는 vitest"라고 다시 말해야 한다면…… 이런 반복 설명은 프로젝트 루트의 `CLAUDE.md`에 한 번만 작성하면 됩니다. 모델이 자동으로 읽으므로, 매 차례 배경을 반복해 설명할 필요가 없습니다.

CLAUDE.md를 더 매끄럽게 쓰는 법은 [CLAUDE.md를 어떻게 써야 잘 쓰는가](/conduit-blog/ko/blog/claude-md-guide/)에 따로 정리했으니 함께 보세요.

## 5. 작은 작업은 Sonnet으로, 걸핏하면 Opus 쓰지 말기

모든 작업에 최강 모델이 필요한 건 아닙니다. 일상적인 소소한 수정, 주석 보강, 테스트 작성에는 Sonnet으로 충분하고, 단가도 훨씬 낮습니다. 정말 복잡한 추론이나 대규모 리팩터링이 필요할 때 Opus로 전환하세요.

전환은 간단합니다.

```bash
/model
```

올바른 모델을 고르는 것 자체가 가장 큰 절약입니다. 상황별로 어떻게 고를지는 [Opus vs Sonnet 어떻게 고를까](/conduit-blog/ko/blog/opus-vs-sonnet/)를 보세요.

## 6. 긴 작업은 작은 단계로 쪼개고, 한꺼번에 밀어넣지 말기

"프로젝트 전체를 한 번에 리팩터링해줘"라고 통째로 던지면, 방대한 파일을 읽고 초장문 응답을 생성해 token이 폭발하고 방향도 어긋나기 쉽습니다. 작은 단계로 쪼개세요.

1. 먼저 리팩터링 계획을 나열하게 시킨다(계획만 출력, 코드는 건드리지 않음)
2. 확인한 뒤, 모듈 하나씩 차례로 수정한다
3. 하나 끝낼 때마다 `/clear` 하고 다음으로 넘어간다

단계로 나누면 token 절약은 물론, 품질도 훨씬 안정적입니다.

## 7. 캐시를 잘 활용해, 반복 컨텍스트를 거듭 보내지 않기

같은 대화 안에서 같은 묶음의 파일을 반복 참조한다면, Claude의 프롬프트 캐시 메커니즘이 반복 부분을 아주 저렴하게 만들어 줍니다. 요령은 **안정된 컨텍스트 구조 유지**입니다. 고정 불변의 배경 정보를 앞에 두고, 변하는 질문을 뒤에 두면 캐시 적중률이 높아집니다.

## 마무리

token 절약의 핵심은 한마디입니다. **모델에게 정말 필요한 것만 주기.** 제때 기록 비우기, 무관한 파일 차단, 파일 정확히 지정, 올바른 모델 선택, 큰 작업 쪼개기. 이 기법들을 조합하니 제 일상 소비가 절반 넘게 줄었고, 효과는 거의 그대로였습니다.
